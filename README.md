# Merlin RNDIS — автоустановщик 5G-модемов для Asuswrt-Merlin

Комплект скриптов для автоматического подключения и поддержания соединения через USB-модемы с интерфейсом **RNDIS** (например, **Fibocom FM350-GL**) на роутерах ASUS с прошивкой **Asuswrt-Merlin**.

## 🚀 Возможности

- Автоматическая инициализация модема (AT-команды, регистрация в сети, активация PDP-контекста).
- Получение IP-адреса, настройка маршрута по умолчанию и DNS.
- Настройка NAT и межсетевого экрана для раздачи интернета в локальную сеть.
- Встроенный **watchdog** для контроля состояния модема и автоматического восстановления.
- Система отложенных перезагрузок (`guard`) при фатальных сбоях.
- Интеграция с событиями прошивки: `wan-event`, `firewall-start`, `nat-start`, `init-start` и др.
- Поддержка нескольких операторов и гибкая настройка предпочитаемых частотных диапазонов (LTE/5G bands).

## 📋 Требования

- Роутер ASUS с прошивкой **Asuswrt-Merlin** (рекомендуется версия 386.x или новее).
- Включённая поддержка **JFFS custom scripts and configs** (Администрирование → Система).
- Настроенный **Dual WAN** с основным интерфейсом `USB`.
- USB-модем с поддержкой RNDIS (протестировано на Fibocom FM350-GL).

## 📥 Установка

### Способ 1: Быстрая установка (рекомендуется)

Подключитесь к роутеру по SSH и выполните:
```bash
cd /tmp
wget https://github.com/jobmail/merlin-rndis/releases/latest/download/scripts.zip
unzip scripts.zip
cd scripts
sh install.sh
```
Если wget или unzip недоступны, используйте curl -L -o scripts.zip <URL> и busybox unzip scripts.zip.

Способ 2: Ручная загрузка

Скачайте архив scripts.zip с последнего релиза.

Скопируйте его в /tmp роутера (например, через scp -O или WinSCP).

Распакуйте и запустите установщик:

```bash
cd /tmp
unzip scripts.zip
sh install.sh
```
⚙️ Настройка

Основные параметры находятся в файле /jffs/scripts/fm350-lib.sh:
```
---------------------------------------------------------------------------------
Переменная	      Значение по умолчанию	    Описание
---------------------------------------------------------------------------------
APN               internet	                Точка доступа вашего оператора
MODEM_TTY         /dev/ttyUSB4	            AT-порт модема
WAN_IF	          eth3	                    Интерфейс, создаваемый модемом
preferredBands	  20,6,3,0                  Предпочитаемые диапазоны (авто‑выбор)
```
Изменить параметры можно вручную в fm350-lib.sh или через аргументы скриптов.

🗑️ Удаление

Для полного удаления проекта выполните в папке с распакованным архивом:
```bash
sh uninstall.sh
```

Скрипт удалит все файлы проекта и уберёт добавленные блоки из событийных скриптов, сохранив остальные пользовательские настройки.

🔄 Обновление

Повторите шаги установки поверх существующей версии. Установщик автоматически заменит старые файлы и обновит событийные скрипты.

📁 Структура проекта
```text
/jffs/scripts/
├── fm350-connect.sh      # основной скрипт соединения
├── fm350-lib.sh          # общие функции и переменные
├── fm350-firewall.sh     # правила FORWARD
├── fm350-nat.sh          # правила NAT (MASQUERADE)
├── fm350-guard.sh        # сторож отложенной перезагрузки
├── set_nvram.sh          # утилита массовой установки NVRAM
├── init.nvram            # начальные параметры NVRAM
└── (событийные скрипты)  # dhcpc-event, wan-event, nat-start и др.
```

🐛 Известные проблемы и их решение

Модем не определяется как /dev/ttyUSB4 — убедитесь, что в настройках Dual WAN выбран USB, а модем подключен напрямую (без хаба) и переключён в режим RNDIS (AT+GTUSBMODE=40).

Ошибка +CME ERROR: 3909 при активации PDP — попробуйте сменить тип APN с IPV4V6 на IP.

Нет интернета в локальной сети — проверьте, что включены NAT и форвардинг. Установщик делает это автоматически, но при конфликтах можно перезапустить фаервол: service restart_firewall.
